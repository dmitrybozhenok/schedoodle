  <action>
    1. Add `isPrivateUrl(urlString: string): boolean` function to `src/services/prefetch.ts`:
       - Import `isIP` from `node:net`.
       - Parse with `new URL(urlString)` in try/catch -- malformed URL returns true (block).
       - Block non-http/https protocols.
       - Check `hostname === 'localhost'` -- block.
       - Check IPv6 loopback: `hostname === '[::1]'` or stripped brackets `=== '::1'` -- block.
       - For IPv4 (check with `isIP(hostname) === 4`): parse octets, check against private ranges:
         - 127.0.0.0/8: octets[0] === 127
         - 10.0.0.0/8: octets[0] === 10
         - 172.16.0.0/12: octets[0] === 172 && octets[1] >= 16 && octets[1] <= 31
         - 192.168.0.0/16: octets[0] === 192 && octets[1] === 168
         - 169.254.0.0/16: octets[0] === 169 && octets[1] === 254
         - 0.0.0.0/8: octets[0] === 0
       - Export the function for testing.

    2. Add response size limiting to `prefetchUrls`:
       - Define `const MAX_RESPONSE_BYTES = 1_048_576;` (1 MB) at module level.
       - Replace the inner fetch logic with a `fetchWithSizeLimit` helper:
         a. After `fetch(url, { signal: AbortSignal.timeout(10_000) })`, check `Content-Length` header. If present and > MAX_RESPONSE_BYTES, return `[Content truncated at 1MB -- ${url}]`.
         b. Otherwise, use streaming reader: `response.body?.getReader()`. Read chunks, accumulate byte count. If total > MAX_RESPONSE_BYTES, cancel reader, return truncation message.
         c. If `response.body` is null, fallback to `response.text()` (still protected by Content-Length check).
         d. Decode chunks with `TextDecoder` and concatenate.
       - After size-limited body is obtained, apply the existing HTML-to-text conversion logic.

    3. Integrate SSRF check into `prefetchUrls`:
       - Before the `fetch()` call inside the `urls.map()` callback, call `isPrivateUrl(url)`.
       - If private, return `{ url, content: '[SSRF blocked -- ${url}]' }` without making any HTTP request.

    4. Create `tests/ssrf.test.ts` for `isPrivateUrl` unit tests:
       - Table-driven tests covering all private ranges, edge cases (172.15 vs 172.16), IPv6, localhost, malformed URLs, non-HTTP protocols, and valid public URLs.

    5. Add tests to `tests/prefetch.test.ts` in a new `describe("SSRF and size limits")` block:

       - Test: prefetchUrls truncates response when Content-Length > 1MB (mock fetch returning Response with content-length header).
       - Test: prefetchUrls truncates response when body stream exceeds 1MB (mock fetch with large body, no Content-Length header -- use a Response with a body that is > 1MB).
       - Test: normal fetch under 1MB still works correctly (existing tests cover this, but add one explicit small-body test).
