<task type="auto" tdd="true">
  <name>Task 2: Create database schema, client, and types</name>
  <files>src/db/schema.ts, src/db/index.ts, src/types/index.ts</files>
  <behavior>
    - Agent can be inserted with name, taskDescription, cronSchedule, systemPrompt
    - Agent can be retrieved by id
    - Agent name uniqueness is enforced case-insensitively ("MyAgent" and "myagent" collide)
    - Execution history can be inserted with all observability fields (agentId, status, inputTokens, outputTokens, durationMs, result as JSON, error, deliveryStatus)
    - Execution history links to agent via foreign key
    - createdAt and updatedAt are auto-populated on agents
    - startedAt is auto-populated on execution_history
  </behavior>
  <action>
1. Create src/db/schema.ts with two tables:

   **agents table:**
   - id: integer primary key autoIncrement
   - name: text, notNull, unique (add COLLATE NOCASE for case-insensitive uniqueness — use `sql` raw if Drizzle column builder does not support collation directly; alternatively create a unique index with `CREATE UNIQUE INDEX agents_name_nocase ON agents(name COLLATE NOCASE)` via a custom migration or after-connect pragma)
   - taskDescription: text('task_description'), notNull
   - cronSchedule: text('cron_schedule'), notNull
   - systemPrompt: text('system_prompt') — nullable
   - createdAt: text('created_at'), notNull, default sql`(CURRENT_TIMESTAMP)`
   - updatedAt: text('updated_at'), notNull, default sql`(CURRENT_TIMESTAMP)`

   **executionHistory table:**
   - id: integer primary key autoIncrement
   - agentId: integer('agent_id'), notNull, references agents.id
   - status: text('status', { enum: ['success', 'failure', 'running'] }), notNull
   - inputTokens: integer('input_tokens') — nullable
   - outputTokens: integer('output_tokens') — nullable
   - durationMs: integer('duration_ms') — nullable
   - result: text('result', { mode: 'json' }) — nullable
   - error: text('error') — nullable
   - deliveryStatus: text('delivery_status') — nullable
   - startedAt: text('started_at'), notNull, default sql`(CURRENT_TIMESTAMP)`
   - completedAt: text('completed_at') — nullable

2. Create src/db/index.ts:
   - Import env from '../config/env.js'
   - Import * as schema from './schema.js'
   - Import path and fs (node:path, node:fs) to ensure data directory exists: `fs.mkdirSync(path.dirname(path.resolve(env.DATABASE_URL)), { recursive: true })` — handles the case where ./data/ does not exist
   - Create drizzle client: `export const db = drizzle(env.DATABASE_URL, { schema })`
   - Enable WAL mode after connection: execute `PRAGMA journal_mode=WAL` via the underlying better-sqlite3 connection (access via `db.$client.pragma('journal_mode = WAL')`)
   - Export Database type: `export type Database = typeof db`

3. Create src/types/index.ts:
   - Import agents and executionHistory types from '../db/schema.js'
   - Export: Agent ($inferSelect from agents), NewAgent ($inferInsert), Execution ($inferSelect from executionHistory), NewExecution ($inferInsert)

IMPORTANT: All relative imports MUST use .js extensions (ESM requirement).
  </action>
