**Modify `src/services/executor.ts`:**

1. Add import: `import { sendNotification } from "./notifier.js";`

2. In the success path of executeAgent, AFTER the existing success DB update (line ~124-134), add notification logic:

```typescript
// --- Notification (fire-and-forget, never affects execution status) ---
try {
  // Set delivery status to pending before send attempt
  db.update(executionHistory)
    .set({ deliveryStatus: "pending" })
    .where(eq(executionHistory.id, executionId))
    .run();

  const notifyResult = await sendNotification(agent.name, new Date().toISOString(), output);

  if (notifyResult.status !== "skipped") {
    db.update(executionHistory)
      .set({ deliveryStatus: notifyResult.status === "sent" ? "sent" : "failed" })
      .where(eq(executionHistory.id, executionId))
      .run();
  }
} catch (err) {
  // Never let notification errors affect execution status
  console.error(`[notify] Unexpected error: ${err}`);
  db.update(executionHistory)
    .set({ deliveryStatus: "failed" })
    .where(eq(executionHistory.id, executionId))
    .run();
}
```

3. The return statement `return { status: "success", executionId, output }` stays AFTER the notification block. The execution result is "success" regardless of notification outcome.

4. Do NOT add notification logic to the failure/catch path -- only successful executions trigger notifications per user decision.
