    - sendNotification returns { status: "sent" } when Resend emails.send succeeds

    - Subject line: "[Schedoodle] {agentName} — {summary}" with summary truncated to 80 chars

    - HTML email contains formatted timestamp in header

    - HTML email contains details section with white-space pre-wrap

    - HTML email omits data section when output.data is undefined

  </behavior>
  <action>
Create `src/services/notifier.ts` following the research patterns:

1. Export `NotifyResult` interface: `{ status: "sent" | "failed" | "skipped"; error?: string }`

2. Create `escapeHtml(str: string): string` helper that replaces &, <, >, " with HTML entities.

3. Create `buildEmailHtml(agentName: string, executedAt: string, output: AgentOutput): string` that renders:
   - DOCTYPE html wrapper with utf-8 charset
   - Inline styles (font-family: system fonts, max-width: 600px, centered)
   - Header div with border-bottom: agent name (h1, 20px) + timestamp (p, 14px, #666)
   - Summary section: h2 "Summary" + paragraph with escaped summary
   - Details section: h2 "Details" + paragraph with escaped details, white-space: pre-wrap
   - Conditional data section: h2 "Data" + pre block with escaped JSON.stringify(output.data, null, 2), background #f4f4f4, padding 12px, border-radius 4px
   - All dynamic content MUST be escaped via escapeHtml before insertion

4. Export `sendNotification(agentName: string, executedAt: string, output: AgentOutput): Promise<NotifyResult>`:
   - Check env.RESEND_API_KEY and env.NOTIFICATION_EMAIL -- if either missing, return { status: "skipped" }
   - Also check env.NOTIFICATION_FROM -- if missing when others are set, return { status: "skipped" } (per research: require all three together to avoid unverified domain errors)
   - Create Resend instance at call time (NOT module level) using env.RESEND_API_KEY
   - Build subject: `[Schedoodle] ${agentName} — ${truncatedSummary}` where truncatedSummary is output.summary sliced to 80 chars with "..." appended if truncated
   - Call resend.emails.send({ from: env.NOTIFICATION_FROM, to: env.NOTIFICATION_EMAIL, subject, html })
   - Destructure { error } from result. If error, log with console.error and return { status: "failed", error: error.message }
   - On success return { status: "sent" }
   - Wrap entire send in try/catch for unexpected errors (network, etc.) -- return { status: "failed" } on catch

- "HTML omits data section when absent" -- output.data undefined, verify no "Data" heading in html
