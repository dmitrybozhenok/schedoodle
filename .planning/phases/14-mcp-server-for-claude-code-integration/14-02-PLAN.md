1. Create `src/mcp/tools/tools.ts` exporting `registerToolTools(server: McpServer, db: Database): void`:
   Register these 8 tools:

   a. **list_tools**: inputSchema `z.object({})` (empty object, no params). Query all tools from DB, return via `jsonResponse`.

   b. **get_tool**: inputSchema `z.object({ id: z.number().describe("Tool ID") })`. Query by ID, 404 with guidance "Use list_tools to see available tools" if not found.

   c. **create_tool**: inputSchema with `name: z.string().describe("Tool name")`, `description: z.string().describe("What the tool does")`, `url: z.string().describe("Webhook URL")`, `method: z.enum(["GET", "POST", "PUT", "PATCH", "DELETE"]).default("POST").describe("HTTP method")`, `headers: z.record(z.string(), z.string()).optional().describe("HTTP headers as key-value pairs")`, `inputSchema: z.record(z.string(), z.any()).describe("JSON Schema for tool input parameters")`. Insert into DB with `.returning().get()`, return via `jsonResponse`.

   d. **update_tool**: inputSchema with `id: z.number()`, all create fields optional. Check exists (404 with guidance), update via drizzle, return updated tool.

   e. **delete_tool**: inputSchema `z.object({ id: z.number().describe("Tool ID to delete"), confirm: z.boolean().default(false).describe("Set to true to confirm deletion") })`. Add `annotations: { destructiveHint: true }`. Same two-step confirmation pattern as delete_agent: preview shows tool details and message "This will permanently delete tool X. Any agents using this tool will lose access. Call delete_tool again with confirm=true to proceed." Confirm=true deletes and returns `{ deleted: true, toolId, toolName }`.

   f. **list_agent_tools**: inputSchema `z.object({ agentId: z.number().describe("Agent ID") })`. Verify agent exists (404 with guidance). Query agentTools join + tools table for this agent's attached tools. Return tool list via `jsonResponse`.

   g. **attach_tool**: inputSchema `z.object({ agentId: z.number().describe("Agent ID"), toolId: z.number().describe("Tool ID to attach") })`. Verify both agent and tool exist (404 with guidance for whichever is missing). Insert into agentTools. Catch UNIQUE constraint for "Tool already attached" error with guidance. Return `{ agentId, toolId, attached: true }`.

   h. **detach_tool**: inputSchema `z.object({ agentId: z.number().describe("Agent ID"), toolId: z.number().describe("Tool ID to detach") })`. Verify the link exists (404 with guidance "Use list_agent_tools to see attached tools"). Delete the link. Return `{ agentId, toolId, detached: true }`.

2. Create `src/mcp/tools/health.ts` exporting `registerHealthTools(server: McpServer, db: Database): void`:
   Register one tool:

   a. **get_health**: inputSchema `z.object({})` (no params). Implement health data aggregation mirroring `src/routes/health.ts` logic but simplified for MCP context:
      - Count agents
      - Query 24h execution history, compute system-wide successRate and avgDurationMs
      - Per-agent breakdown: lastRunAt (from latest execution), lastStatus, successRate (24h), avgDurationMs (24h), healthy (consecutiveFailures < 3), consecutiveFailures
      - Import `getLlmCircuitStatus` from executor for circuit breaker status
      - Import `getLlmSemaphoreStatus` from executor for concurrency stats
      - Top-level status: OPEN circuit breaker = "unhealthy", >50% agents unhealthy = "unhealthy", some = "degraded", none = "ok"
      - NOTE: No upcomingRuns (MCP server has no scheduler, so no job registry available). Include a note in the response: `upcomingRuns: "Not available from MCP server (scheduler runs in HTTP server process)"`.
      - Return full health response via `jsonResponse`.
