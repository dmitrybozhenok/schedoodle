3. Create `src/mcp/helpers.ts` with shared response formatting:
   - `jsonResponse(data: unknown)` returns `{ content: [{ type: "text", text: JSON.stringify(data, null, 2) }] }`
   - `errorResponse(error: string, guidance: string)` returns `{ content: [{ type: "text", text: JSON.stringify({ error, guidance }, null, 2) }], isError: true }`
   - Export both functions.

4. Create `src/mcp/tools/agents.ts` exporting `registerAgentTools(server: McpServer, db: Database): void`:
   Register these 5 tools using `server.registerTool()` with Zod v4 inputSchemas:

   a. **list_agents**: inputSchema `z.object({ enabled: z.enum(["true", "false"]).optional().describe("Filter by enabled status") })`. Query agents table, apply enabled filter if provided (convert string to integer 0/1), map through `enrichAgent(agent, db)`, return via `jsonResponse`.

   b. **get_agent**: inputSchema `z.object({ id: z.number().describe("Agent ID") })`. Query by ID, 404 with guidance if not found, return `enrichAgent(agent, db)` via `jsonResponse`.

   c. **create_agent**: inputSchema with `name: z.string().describe("Agent name")`, `taskDescription: z.string().describe("What the agent should do")`, `cronSchedule: z.string().describe("Cron expression or natural language schedule (e.g., 'every weekday at 9am')")`, `systemPrompt: z.string().optional().describe("System prompt to shape LLM behavior")`, `model: z.string().optional().describe("LLM model ID")`, `enabled: z.boolean().default(true).describe("Whether the agent should be enabled")`. Handler: if cronSchedule is not already a cron expression (use `isCronExpression()`), call `parseSchedule()` to convert NL to cron. Insert into DB with `.returning().get()`. Do NOT call `scheduleAgent()` (MCP server has no scheduler per locked decision). Return enriched agent via `jsonResponse` with 201-equivalent message. Catch UNIQUE constraint for 409-style error with guidance.

   d. **update_agent**: inputSchema with `id: z.number()`, all create fields optional. Same NL schedule resolution. Update via drizzle `.update().set().where().returning().get()`. Do NOT call `scheduleAgent()`/`removeAgent()`. Return enriched result.

   e. **delete_agent**: inputSchema `z.object({ id: z.number().describe("Agent ID to delete"), confirm: z.boolean().default(false).describe("Set to true to confirm deletion. Default shows preview only.") })`. Add `annotations: { destructiveHint: true }`. When `confirm=false` (default): look up agent, return preview with enriched data and message "This will permanently delete agent X and all its execution history. Call delete_agent again with confirm=true to proceed." When `confirm=true`: delete agent from DB (cascade handles agent_tools), return `{ deleted: true, agentId, agentName }`. 404 with guidance if agent not found.

   f. **execute_agent**: inputSchema `z.object({ id: z.number().describe("Agent ID to execute") })`. Look up agent, check enabled status (return error with guidance "Enable the agent first using update_agent" if disabled, mirroring REST API 409 behavior). Call `executeAgent(agent, db)` directly (synchronous, may take 10-60s). Return the full ExecuteResult via `jsonResponse`.

   IMPORTANT: All tools must use `.describe()` on each inputSchema field for good tool documentation. Use `console.error()` for any logging (NEVER `console.log()` -- stdout is the MCP transport channel).

5. Create `src/mcp/tools/history.ts` exporting `registerHistoryTools(server: McpServer, db: Database): void`:
   Register one tool:

   a. **get_execution_history**: inputSchema `z.object({ agentId: z.number().describe("Agent ID"), limit: z.number().default(100).describe("Max results (default 100, max 200)") })`. Verify agent exists (404 with guidance if not). Query executionHistory ordered by startedAt DESC, limit capped at 200. Return results via `jsonResponse`.

6. Create `src/mcp.ts` entrypoint:
   ```typescript
   import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";
   import { StdioServerTransport } from "@modelcontextprotocol/sdk/server/stdio.js";
   import { db } from "./db/index.js";
   import { registerAgentTools } from "./mcp/tools/agents.js";
   import { registerHistoryTools } from "./mcp/tools/history.js";

   const server = new McpServer({
     name: "schedoodle",
     version: "1.0.0",
   });

   registerAgentTools(server, db);
   registerHistoryTools(server, db);
   // Plan 02 will add: registerToolTools, registerHealthTools, registerScheduleTools

   async function main() {
     const transport = new StdioServerTransport();
     await server.connect(transport);
     console.error("[mcp] Schedoodle MCP server running on stdio");
   }

   main().catch((error) => {
     console.error("Fatal error in MCP main():", error);
     process.exit(1);
   });
   ```
