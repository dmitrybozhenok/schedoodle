- Mock executeAgent from src/services/executor.ts
- Mock the database (use a minimal mock that returns agent rows)
- Test scheduleAgent: create a Cron job with a fast pattern (every second "* * * * * *" for croner), verify executeAgent is called within ~1.5s, then stop
- Test removeAgent: schedule then remove, verify job stopped (use jobs Map size or croner isStopped)
- Test startAll: pass array of agents, verify all scheduled
- Test stopAll: schedule agents then stopAll, verify all stopped

- Test stale data avoidance: verify scheduler re-reads agent from DB before executing (the closure should query DB by agent ID, not use the captured agent object)

Create src/services/scheduler.ts:
- `const jobs = new Map<number, Cron>()`
- `scheduleAgent(agent: Agent, db: Database)`: Remove existing job if any. Create new Cron with agent.cronSchedule. In the callback: re-read agent from DB using `db.select().from(agents).where(eq(agents.id, agentId)).get()`. If agent not found (deleted between schedule and trigger), log warning and return. Otherwise call executeAgent with fresh agent data. Log `[cron] Executing: {name}` before and `[cron] {name}: {status} in {elapsed}s` after.
- `removeAgent(agentId: number)`: Get job from Map, call job.stop(), delete from Map. No-op if not found.
- `startAll(agentList: Agent[], db: Database)`: Call scheduleAgent for each agent. Log `[cron] Scheduled {N} agent(s)`.
- `stopAll()`: Iterate Map, stop each job, clear Map.
- Export all four functions plus `getJobCount()` for testing.

Use croner's `{ name: "agent-{id}" }` option for job naming.
