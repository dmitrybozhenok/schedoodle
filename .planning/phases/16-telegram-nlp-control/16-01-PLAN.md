**1. Create `src/schemas/telegram-intent.ts`:**

```typescript
import { z } from "zod";

export const telegramIntentSchema = z.object({
  action: z.enum([
    "list",
    "run",
    "enable",
    "disable",
    "status",
    "reschedule",
    "unknown",
  ]).describe("The user's intended action"),
  agentName: z.string().nullable()
    .describe("Exact agent name from the provided list, or null if action doesn't target a specific agent (list, status, unknown)"),
  scheduleInput: z.string().nullable()
    .describe("The natural language schedule text for reschedule action, or null for other actions"),
});

export type TelegramIntent = z.infer<typeof telegramIntentSchema>;
```

**2. Create `src/services/intent-parser.ts`:**

- Import `generateText`, `NoObjectGeneratedError`, `Output` from "ai"
- Import `z` from "zod"
- Import `DEFAULT_MODEL`, `resolveModel` from "../config/llm-provider.js"
- Import `telegramIntentSchema`, `TelegramIntent` from "../schemas/telegram-intent.js"

- Export async function `parseIntent(userMessage: string, agentNames: string[]): Promise<TelegramIntent>`
  - Build agent list string: `agentNames.map((n, i) => \`${i + 1}. ${n}\`).join("\n")` -- or `"(no agents configured)"` if list is empty
  - Build system prompt:
    ```
    You are a command parser for a bot that controls scheduled agents.

    Available agents:
    ${agentList}

    Parse the user's message and extract their intent.

    Rules:
    - Match agent names fuzzily: "briefing" should match "Morning Briefing Agent", "PR" should match "PR Reminder"
    - For "list" and "status" actions, set agentName to null
    - For "reschedule", extract the schedule description into scheduleInput (e.g., "change briefing to 8am" -> scheduleInput: "8am")
    - For "run", "enable", "disable", set agentName to the EXACT name from the agent list above
    - If you cannot determine the intent, use action "unknown"
    - If the user mentions an agent that doesn't exist in the list, use action "unknown"
    ```
  - Call `resolveModel(DEFAULT_MODEL)` to get the model
  - Call `generateText({ model, output: Output.object({ schema: telegramIntentSchema }), prompt: \`User message: "${userMessage}"\`, system: systemPrompt })`
  - On success, return `result.output as TelegramIntent`
  - On `NoObjectGeneratedError`, retry once with error feedback appended to the prompt (matching schedule-parser.ts retry pattern exactly):
    ```typescript
    const errorMsg = error instanceof Error ? error.message : String(error);
    const retryPrompt = `User message: "${userMessage}"\n\n[Previous attempt failed: ${errorMsg}]\nPlease provide a valid response matching the required schema.`;
    ```
  - Re-throw all other errors

**1. Create `src/services/telegram-poller.ts`:**

Define Telegram API types at the top of the file:

```typescript
export interface TelegramUpdate {
  update_id: number;
  message?: TelegramMessage;
}

export interface TelegramMessage {
  message_id: number;
  from?: { id: number; first_name: string };
  chat: { id: number; type: string };
  text?: string;
  date: number;
}

interface TelegramGetUpdatesResponse {
  ok: boolean;
  result: TelegramUpdate[];
  description?: string;
}
```

Module-level state:

```typescript
let running = false;
```

Export function `startPolling(botToken: string, chatId: string, onMessage: (msg: TelegramMessage) => Promise<void>): void`
- Set `running = true`
- Initialize `let offset = 0`
- Define inner `async function poll()`:
  - `while (running)` loop:
    - try block:
      - `fetch(\`https://api.telegram.org/bot${botToken}/getUpdates\`, { method: "POST", headers: { "Content-Type": "application/json" }, body: JSON.stringify({ offset, timeout: 30, allowed_updates: ["message"] }) })`
      - Parse response as `TelegramGetUpdatesResponse`
      - If `data.ok && data.result.length > 0`:
        - for-of loop over `data.result`:
          - Always update `offset = update.update_id + 1` (even before processing, to avoid reprocessing on error)
          - If `update.message?.text`:
            - Security check: `if (String(update.message.chat.id) !== chatId)` -> `continue` (silent ignore, no log, no response)
            - `await onMessage(update.message)` (sequential processing within batch)
    - catch block:
      - `console.error(\`[telegram-bot] Polling error: ${err instanceof Error ? err.message : err}\`)`
      - `if (running) await new Promise(r => setTimeout(r, 5000))` (5 second backoff on error, but only if still running)
- Call `void poll()` (fire-and-forget, starts the loop)

Export function `stopPolling(): void`
- Set `running = false`

Export function `isPollingActive(): boolean`
- Return `running`

Also export a plain-text reply helper (research recommends no MarkdownV2 for bot control responses):

```typescript
export async function sendPlainText(botToken: string, chatId: string | number, text: string): Promise<void> {
  const url = `https://api.telegram.org/bot${botToken}/sendMessage`;
  await fetch(url, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ chat_id: String(chatId), text }),
  });
}
```

Export a typing indicator helper:

```typescript
export async function sendTypingAction(botToken: string, chatId: string | number): Promise<void> {
  const url = `https://api.telegram.org/bot${botToken}/sendChatAction`;
  await fetch(url, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ chat_id: String(chatId), action: "typing" }),
  });
}
```
