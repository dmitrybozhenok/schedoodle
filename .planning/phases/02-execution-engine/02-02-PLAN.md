    - Test 12: executeAgents -- one agent failing does not prevent other agents from completing successfully

  <action>
    Create `src/services/executor.ts` implementing:

    1. `executeAgent(agent: Agent, db: Database)` -- the core function:
       - Insert a `{ agentId: agent.id, status: "running" }` record into executionHistory using `.returning()` to get the execution ID
       - Call `prefetchUrls(agent.taskDescription)` to get context data
       - Call `buildPrompt(agent.taskDescription, contextData)` to build the user message
       - Call `callLlmWithRetry()` (internal helper) with the agent's model, system prompt, and user message
       - On success: update execution record with status "success", result (the parsed output), inputTokens, outputTokens (use `?? null` since they may be undefined), durationMs, completedAt
       - On failure: update execution record with status "failure", error message, durationMs, completedAt
       - Return `{ status, executionId, output }` or `{ status, executionId, error }`
       - IMPORTANT: Use `await` on all DB operations that use `.returning()` or `.run()` -- Drizzle with better-sqlite3 returns synchronously but the types may expect await

    2. `callLlmWithRetry(model, systemPrompt, userMessage)` -- internal helper:
       - First attempt: `generateText({ model: anthropic(model), system: systemPrompt ?? undefined, output: Output.object({ schema: agentOutputSchema }), prompt: userMessage })`
       - If `NoObjectGeneratedError.isInstance(error)`: retry once with validation error appended to prompt as feedback
       - If second attempt also fails: throw the error (caller catches it)
       - Non-validation errors: throw immediately (no retry)
       - Default model: `agent.model ?? "claude-sonnet-4-20250514"`

    3. `executeAgents(agents: Agent[], db: Database)` -- concurrent wrapper:
       - `Promise.allSettled(agents.map(a => executeAgent(a, db)))`
       - Return the settled results array

    Create `tests/executor.test.ts`:
    - Mock `"ai"` module: `vi.mock("ai", ...)` to intercept generateText. Return a fake result with `{ output: { summary: "test", details: "test" }, usage: { inputTokens: 10, outputTokens: 20 } }`.
    - Mock `"@ai-sdk/anthropic"` module: `vi.mock("@ai-sdk/anthropic", ...)` to return a mock anthropic function.
    - Mock `"../services/prefetch.js"` (or use the real module with mocked fetch -- either approach works; mocking prefetch is simpler since it's tested separately).
    - Use an in-memory SQLite database (`:memory:`) for DB assertions. Set up the schema with drizzle-kit push or manual table creation.
    - For NoObjectGeneratedError tests: create a mock error object where `NoObjectGeneratedError.isInstance()` returns true, or mock the isInstance static method.
    - For concurrency test: mock generateText to resolve after a small delay for one agent and reject for another; verify both complete.

    IMPORTANT anti-patterns to avoid:
    - Do NOT use `generateObject` -- it is deprecated in AI SDK 6. Use `generateText` + `Output.object()`.
    - Do NOT use `usage.promptTokens` or `usage.completionTokens` -- AI SDK uses `usage.inputTokens` and `usage.outputTokens`.
    - Do NOT forget `.run()` on Drizzle update queries -- without it the update never executes.
    - Import with `.js` extension for local imports (ESM project).
