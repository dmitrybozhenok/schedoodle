    - Test 6: prefetchUrls passes JSON response as raw string (mocked fetch returning JSON with content-type application/json)
    - Test 7: prefetchUrls returns failure note on fetch error (mocked fetch that rejects)
    - Test 8: prefetchUrls returns failure note on timeout (mocked fetch with AbortSignal timeout)
    - Test 9: buildPrompt returns task description unchanged when no context data
    - Test 10: buildPrompt appends pre-fetched data sections to task description
  </behavior>
  <action>
    Create `src/services/prefetch.ts` with three exported functions:

    1. `extractUrls(text: string): string[]` -- Use regex `/https?:\/\/[^\s)>\]]+/g` to find URLs in text, deduplicate with `new Set()`, return as array.

    2. `prefetchUrls(taskDescription: string): Promise<Map<string, string>>` -- Extract URLs from task description, fetch each with `AbortSignal.timeout(10_000)`, check content-type header:
       - `application/json` -> return raw body text
       - `text/html` -> convert with `convert()` from `html-to-text` (use `{ wordwrap: 120 }` option)
       - anything else -> return raw body text as plain text fallback
       On fetch failure: catch error, set `results.set(url, "[Failed to fetch ${url} -- ${reason}]")`. Use `Promise.allSettled` to fetch all URLs concurrently.

    3. `buildPrompt(taskDescription: string, contextData: Map<string, string>): string` -- If contextData is empty, return taskDescription unchanged. Otherwise append each entry as `--- Content from ${url} ---\n${content}\n--- End ---` sections under a "Pre-fetched reference data:" header.
